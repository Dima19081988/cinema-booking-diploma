# 01. Требования

## 2. Роли пользователей

В системе предусмотрены две роли:
- гость;
- администратор.

## 3. Функциональные требования

### 3.1. Возможности гостя

- просмотр списка фильмов;
- просмотр информации о фильме;
- просмотр расписания сеансов;
- выбор даты сеанса;
- просмотр схемы кинозала;
- выбор свободного места;
- бронирование билета на конкретную дату.

### 3.2. Возможности администратора

- создание или редактирование залов.
- создание или редактирование списка фильмов.
- настройка цен.
- создание или редактирование расписания сеансов фильмов.
- просмотр броней.
- изменение данных системы.

## 4. Сущности системы

В системе должны быть реализованы следующие сущности:

- кинозал. 
Помещение, в котором демонстрируются фильмы. Режим работы определяется расписанием на день. Зал — прямоугольное помещение, состоит из N х M различных зрительских мест.

- зрительское место. 
Место в кинозале. Есть два вида: VIP и обычное.
Фильм. Информация о фильме заполняется администратором. Фильм связан с сеансом в кинозале.

- фильм.
Информация о фильме заполняется администратором. Фильм связан с сеансом в кинозале.

- сеанс. 
Временной промежуток, во время которого в кинозале будет показываться фильм. На сеанс могут быть забронированы билеты.

- билет.
QR-код c уникальным кодом бронирования, в котором обязательно указаны место, ряд, сеанс. Билет действителен строго на свой сеанс. Для генерации QR-кода можно использовать библиотеку.

- пользователь.
Учетная запись пользователя системы, ролевая модель подразумевает наличие слкдующих ролей: Администратор — авторизованный пользователь, Гость — неавторизованный посетитель сайта.

## 5. Требования к билетам

Билет должен содержать:
- QR-код с уникальным кодом бронирования;
- информацию о месте;
- информацию о ряде;
- информацию о сеансе;
Билет действителен строго на свой сеанс.

## 6. Требования к безопасности

- серверная валидация входных данных;
- хранение паролей в хешированном виде;
- проверку хеша пароля при аутентификации;
- разграничение прав доступа между гостем и администратором.

## 7. Технологические требования

### 7.1. Backend

- Django + Django REST Framework;
- СУБД (PostgreSQL).

### 7.2. Frontend

- JavaScript;
- HTML;
- CSS;
- React;
- React Router.
В работе над интерфейсом используются собственные UI-компоненты системы.
